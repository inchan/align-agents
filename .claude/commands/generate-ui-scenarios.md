# UI 시나리오 테스트 생성

UI 기준으로 시나리오 테스트를 생성합니다.

## 인자

- `$ARGUMENTS`: 대상 기능/페이지 설명 (예: "Rules 관리 페이지", "MCP 서버 설정", "사용자 인증")

## 작업 지시사항

### 1단계: 컨텍스트 분석

1. **대상 기능 파악**: `$ARGUMENTS`에서 테스트 대상 기능을 파악하세요.

2. **관련 코드 탐색**: 해당 기능과 관련된 컴포넌트, 페이지, API를 찾으세요.
   - 컴포넌트: `packages/web/src/` 또는 `src/` 하위
   - API: `packages/api/` 또는 관련 서비스 파일

3. **기존 시나리오 검토**: `./docs/e2e-tests/scenarios/` 디렉토리의 기존 시나리오 파일들을 확인하여 형식과 패턴을 파악하세요.

### 2단계: UI 시나리오 생성

다음 카테고리별로 시나리오를 작성하세요:

| 카테고리 | 우선순위 | 설명 |
|---------|---------|------|
| **Core** | P0 | 핵심 CRUD 기능, 필수 통과 |
| **Validation** | P0 | 입력 유효성 검증 |
| **Error** | P1 | 에러 처리, 네트워크 실패 |
| **Edge** | P1-P2 | 경계 조건, 특수 상황 |
| **Accessibility** | P2 | 키보드 네비게이션, 스크린리더 |

### 3단계: 시나리오 파일 작성

`./docs/e2e-tests/scenarios/` 디렉토리에 저장:
- 파일명: `[기능명-kebab-case].md` (예: `rules.md`, `mcp-servers.md`)
