# 웹 UI 개선 계획

## 🔍 현재 상태 분석

### ✅ 작동하는 기능

- Dashboard: 도구 목록 및 통계 표시
- Tools 페이지: 도구 목록 표시, 정렬, 툴팁
- Rules 페이지: 조회, 편집, 저장, 동기화 (기본)
- MCP 페이지: 조회, 편집, 저장, 동기화 (기본)
- API 연동: 모든 엔드포인트 정상 작동

### ❌ 안되거나 미완성인 기능

#### UI/UX 문제

1. **Alert 남용**: 모든 메시지가 alert로 표시 (토스트 알림 필요)
2. **로딩 상태 부족**: 로딩 인디케이터 없음
3. **에러 표시 부족**: 에러 메시지가 간단한 alert만
4. **동기화 결과 부족**: "3/4 성공"만 표시, 상세 정보 없음
5. **폼 유효성 검증 부족**: 입력 검증 미흡
6. **반응형 디자인 미흡**: 모바일 최적화 부족

#### 기능 문제

1. **MCP 서버 관리**: JSON 직접 편집만 가능 (추가/삭제 UI 없음)
2. **도구 설정 편집**: "설정 편집" 버튼이 alert만 표시
3. **Rules 템플릿**: 템플릿 선택 UI 없음
4. **백업/복원**: UI 없음 (CLI만 가능)
5. **히스토리 관리**: UI 없음 (CLI만 가능)
6. **동기화 진행 상태**: 진행률 표시 없음
7. **실시간 업데이트**: 수동 새로고침 필요

---

## 🎯 우선순위별 개선 계획

### 🔴 즉시 개선 (High Priority) - 1-2일

#### 1. 토스트 알림 시스템 (2-3시간)

**현재**: alert 사용  
**개선**: react-hot-toast 또는 sonner 사용  
**효과**: 사용자 경험 대폭 개선

```typescript
// 예시
toast.success('Rules가 저장되었습니다');
toast.error('동기화 실패: 네트워크 오류');
toast.loading('동기화 중...');
```

#### 2. 로딩 인디케이터 (1시간)

**현재**: "Loading..." 텍스트만  
**개선**: 스피너, 스켈레톤 UI  
**효과**: 로딩 상태 명확히 표시

#### 3. MCP 서버 추가/삭제 UI (3-4시간)

**현재**: JSON 직접 편집  
**개선**: 폼 기반 추가/삭제 UI  
**효과**: 사용자가 쉽게 MCP 서버 관리

```
[+ 서버 추가] 버튼
→ 모달: 이름, 명령어, 인자, 환경변수 입력
→ 저장 → 목록에 추가
```

#### 4. 동기화 결과 상세 표시 (2-3시간)

**현재**: "3/4 성공" alert  
**개선**: 모달 또는 패널로 상세 결과 표시  
**효과**: 어떤 도구가 성공/실패했는지 명확히 표시

```
동기화 결과
✅ Claude Desktop: 성공
✅ Codex: 성공
✅ Claude Code CLI: 성공
❌ Gemini CLI: 실패 - 파일 권한 오류
```

#### 5. 에러 처리 개선 (1-2시간)

**현재**: 간단한 에러 메시지  
**개선**: 상세 에러 정보, 해결 방법 제시  
**효과**: 사용자가 문제 해결 가능

---

### 🟡 단기 개선 (Medium Priority) - 2-3일

#### 6. 도구 설정 편집 모달 (4-5시간)

**현재**: alert만 표시  
**개선**: 실제 설정 파일 편집 가능한 모달  
**효과**: 웹 UI에서 도구 설정 직접 수정

#### 7. Rules 템플릿 선택 UI (2-3시간)

**현재**: 없음  
**개선**: 템플릿 목록 표시 및 적용 버튼  
**효과**: 빠르게 Rules 설정 가능

```
템플릿 선택:
○ React
○ Vue
○ Node.js
○ Python
[적용]
```

#### 8. 폼 유효성 검증 (2시간)

**현재**: 최소한의 검증  
**개선**: 실시간 유효성 검증, 에러 메시지  
**효과**: 잘못된 입력 방지

#### 9. 반응형 디자인 개선 (3-4시간)

**현재**: 데스크톱 위주  
**개선**: 모바일, 태블릿 최적화  
**효과**: 모든 기기에서 사용 가능

---

### 🟢 장기 개선 (Low Priority) - 3-5일

#### 10. 백업/복원 UI (4-6시간)

**현재**: CLI만 가능  
**개선**: 웹 UI에서 백업 생성, 목록 보기, 복원  
**효과**: 웹에서 완전한 관리 가능

#### 11. 히스토리 관리 UI (4-6시간)

**현재**: CLI만 가능  
**개선**: 변경 히스토리 보기, 롤백  
**효과**: 변경 사항 추적 및 복구 가능

#### 12. 실시간 업데이트 (6-8시간)

**현재**: 동기화 완료 시 `invalidateQueries`로 갱신 (polling 제거됨)
**개선**: WebSocket으로 실시간 알림 및 자동 갱신
**효과**: 외부 변경(CLI 직접 실행 등) 감지 가능

#### 13. 다크 모드 (2-3시간)

**현재**: 라이트 모드만  
**개선**: 다크 모드 지원  
**효과**: 사용자 선호도 지원
